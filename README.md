### 회고
이번 프로젝트는 본격적인 협업의 맛을 보았다. 이전까지의 협업은 단순 소통이었다. 팀원들간의 생각을 동일하게 맞추어 우리가 같은 곳을 바라보는 것 정도. 
하지만 이번 프로젝트에서의 협업은 단순 소통이 아니었다. 다행스럽게도 프로젝트 주제와 토대가 되는 내용을 관호님이 정리하여 오셨기 때문에 이전처럼 생각을 맞추는 일이 필요하지 않았다. 하지만 도메인끼리의 상호작용이라는 장애물이 있었다. 모놀리식으로 개발을 진행할 때에도 있었지만 서버를 MSA 형식으로 분리하자 장애가 본격화되었다. 
내가 맡았던 Order 도메인은 다른 모든 도메인에서 데이터를 받아와 최종적으로 주문을 생성해야 했다. 모놀리식에서 내가 구현해두었던 Redis 재고 차감이나 값의 유효성 검증 등의 로직들은 모두 다른 도메인으로 넘어갔고 대부분의 작업이 Api 요청 후 해당 값의 유효성 검증과 주문을 생성하는 것이었다. 그 과정에서 객체 지향적인 코드를 작성하는 것을 목표로 하고 각 클래스들을 분리하고 정리하는 작업을 거쳤다. 그리고 다른 도메인들의 기능 구현과 테스트가 끝나고 테스트를 진행하여 발생하는 오류들을 수정했다. 그 뒤에도 필수 산출물들을 만들고 추가로 필요한 로직들을 구현하며 프로젝트를 무사히 마칠 수 있었다. 
이번 프로젝트를 진행하며 가장 크게 느낄 수 있었던 점들은 왜 코드를 객체 지향적으로 작성하는 가와 협업에서의 소통의 중요성이었다. 
우선 객체 지향적으로 코드를 작성해야 한다고 생각한 이유는 수정이나 로직을 추가하는 것에 있어 굉장히 간편하기 때문이었다. 초기에는 시간이 조금 더 필요한 것은 사실이지만 수정 또는 확장에 매우 편리했다. 또한 서비스 내의 코드가 흐름 별로 메소드만 깔끔하게 작성되어 있어 코드의 가독성도 매우 뛰어났다. 이론으로만 접하였을 때에는 와닿지 않았지만 이번 프로젝트 경험을 통해 OOP의 필요성을 뼈저리게 느낄 수 있었다. 
다음으로 협업에 있어 소통은 두 말 필요없이 중요했다. 내게 필요한 API 작성을 요청하는 것, 그 과정에서 팀원들과 소통을 거듭하며 코드의 양식을 맞추어야 했고, 오류가 발생한 부분에서도 소통을 통해 문제점을 발견하고 해결해야 했다. 앞서 말했듯이 이전 프로젝트에서의 소통은 단순 생각을 맞추는 일이었지만 이번 프로젝트에서의 소통은 개발에 직접적으로 관여되는 영역이었다. 
이번 프로젝트는 분명 쉽지 않았고, 일정 역시 빠듯했지만 얻어가는 것이 매우 많은 프로젝트였다. 부족한 부분들을 찾는 시간이자 새로운 것을 배우는 시간이었다.

# 📌 프로젝트 개요 (TimeDeal)

TimeDeal은 관리자가 선택한 특정 상품을 제한된 시간 동안 특별한 조건으로 판매하는 프로모션 서비스다.
상품은 사전에 등록되어 있으며, 관리자가 이 중 일부를 선택해 프로모션을 생성한다.
프로모션이 시작되면 사용자에게 알림이 전송되어 빠르게 접근할 수 있도록 도와주며, 동시에 많은 사용자가 몰리는 상황을 대비해 대기열 시스템을 적용해 서버 부하를 완화한다.

구매 과정에서는 락(lock) 처리를 적용하여 과도한 요청으로 인한 재고 오류를 방지하며, 사용자는 주문 완료 시 재고 차감과 함께 **얼마나 가격을 절약했는지(Save 금액)**를 확인할 수 있다.

---

# 🎯 주요 목표

* 관리자 중심의 프로모션 운영

  * 사전 등록된 상품을 기반으로 관리자가 원하는 상품을 선정해 프로모션 생성.

  * 프로모션 기간/재고/할인율 등 조건을 유연하게 설정.

* 사용자 알림 및 접근성 향상

  * 프로모션 시작 시 사용자에게 즉각 알림 발송.

  * 사용자가 빠르게 참여할 수 있도록 UI·UX 개선.

* 안정적인 구매 환경 제공

  * 대량 트래픽 발생 시 대기열 시스템으로 서버 안정성 확보.

  * 경쟁 상황에서 정확한 주문 처리를 위해 락 기반 재고 관리 적용.

* 정확한 재고 및 주문 처리

  * 주문 시 즉시 재고 차감.
  * 초과 주문, 중복 처리 등을 방지하여 신뢰도 높은 거래 환경 제공.

* 사용자 혜택 시각화

  * 구매 후 할인에 따른 **절약 금액(Save Amount)**을 표시하여 만족도 증가.

# 기대효과

대기열 시스템과 동시성 제어를 통해 트래픽이 몰리는 상황에서도 안정적인 서비스 운영이 가능해진다.

락 기반 재고 관리로 초과 주문이나 재고 불일치 문제가 발생하지 않아 신뢰도 높은 주문 처리가 이루어진다.

관리자가 프로모션 상품을 직접 선정하고 설정할 수 있어 운영 효율성과 관리 편의성이 향상된다.

프로모션 알림과 손쉬운 접근성을 통해 사용자 참여율이 높아진다.

할인 혜택과 절약 금액(Save Amount)을 시각적으로 보여줌으로써 사용자 만족도가 크게 증가한다.
